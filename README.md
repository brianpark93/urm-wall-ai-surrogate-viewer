# URM Wall AI Surrogate Viewer

Interactive web app for an L-shaped unreinforced masonry (URM) long wall:
type or slide to any peak ground acceleration (PGA) and see the predicted
crack pattern from two trained neural-network surrogates, side by side.

Open `index.html` (served over HTTP, e.g. via GitHub Pages, or locally with
`python -m http.server`) — no build step, no server-side component. Both
models were trained once (offline, PyTorch) on 185 IDA runs of the wall and
exported as plain JSON weight files; the forward pass for whatever PGA you
enter runs live in this page's JavaScript.

## Panels

| Panel | Model |
|-------|-------|
| Left  | Plain MLP + Fourier positional encoding, `f(PGA, x, y, z) -> damage` |
| Right | Same architecture + a physics-informed monotonicity penalty (damage should not decrease as PGA increases), i.e. a lightweight PINN |

- PGA above 0.66g is flagged as **extrapolation** — beyond every simulated
  training run.
- The physics-informed model is **not** shown here because it's more
  accurate — leave-one-out testing on the underlying dataset found the
  opposite (PCA+GPR and the plain MLP both beat it once real damage
  develops, since this specific wall's damage-state transitions are
  documented as non-monotonic). It's included to compare what the
  constraint changes qualitatively, especially on extrapolation behavior.
- Isometric projection (az=45°, el=25°), same convention used throughout
  the parent research project. White = intact, black = fully
  damaged/eroded. Long-wall mortar elements only (8,480).

## Data / model provenance

Generated by `export_web_model.py` in the parent research repository
(L-shaped masonry seismic fragility study, KAIST). Not included here —
this repo only ships the trained weights (`data/model_plain.json`,
`data/model_pinn.json`) and precomputed element geometry
(`data/geometry_lw.json`), not the training pipeline or raw FE simulation
data.
